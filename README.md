# Gitlab_talkies

An end-to-end Retrieval-Augmented Generation (RAG) system for querying the GitLab company handbook. Ask plain-English questions and get accurate, sourced answers drawn directly from handbook content — complete with a user authentication system, conversation threads, and a two-layer safety guardrail.

## Overview

This project lets any registered user ask questions about the GitLab handbook and receive contextual answers powered by a local embedding model (BGE) and Google Gemini. All questions and answers are persisted in MongoDB inside user-owned conversation **threads**, enabling full conversation history.

Key capabilities:

- **RAG pipeline** — ChromaDB vector store + BGE embeddings + Gemini LLM
- **Two-layer guardrails** — logical (instant) + prompt-based (LLM classifier) safety checks
- **REST API** — Flask backend with session-based auth and full thread/query CRUD
- **Streamlit UI** — clean, modular frontend that talks to the Flask API over HTTP
- **Per-user isolation** — every thread and query is scoped to the authenticated user

## Tech Stack

| Layer | Technology |
|---|---|
| Embedding model | `BAAI/bge-small-en-v1.5` via `sentence-transformers` |
| Vector store | ChromaDB (`langchain-chroma`) |
| LLM — answers | Google Gemini 2.5 Flash (`langchain-google-genai`) |
| LLM — guardrails | Google Gemini 2.0 Flash |
| Backend framework | Flask 3 + Flask-CORS |
| Database | MongoDB (`pymongo`) |
| Auth | Server-side sessions, bcrypt password hashing (`werkzeug`) |
| Frontend | Streamlit 1.35+ |
| Frontend → Backend | `requests` (persistent session for cookie forwarding) |
| Config | `python-dotenv` |

## Prerequisites

Ensure the following are installed on your machine before proceeding:

- **Python 3.11+**
- **MongoDB** — running locally on port 27017, or a MongoDB Atlas connection string
- **Google API Key** — with Gemini API access enabled ([get one here](https://aistudio.google.com/app/apikey))
- **ChromaDB vector store** — the `chroma_db/` directory must already be populated by your `vector_db_builder.py` script before starting the backend

## Installation

**1. Clone the repository**

```bash
git clone https://github.com/your-org/gitlab-handbook-rag.git
cd gitlab-handbook-rag
```

**2. Create and activate a virtual environment**

```bash
python -m venv .venv

# macOS / Linux
source .venv/bin/activate

# Windows
.venv\Scripts\activate
```

**3. Install all dependencies**

```bash
pip install -r requirements.txt

## Running the Project

The backend and frontend are **two separate processes**. You need two terminal windows.

### Terminal 1 — Flask Backend

```bash
# Activate your venv first
source .venv/bin/activate

python main.py
```

Expected output:
```
 * Running on http://0.0.0.0:5000
 * Debug mode: off
```

### Terminal 2 — Streamlit Frontend

```bash
# Activate your venv first
source .venv/bin/activate

streamlit run streamlit_app/app.py
```

Expected output:
```
  You can now view your Streamlit app in your browser.
  Local URL: http://localhost:8501
```

Open **http://localhost:8501** in your browser. Register an account, log in, create a thread, and start asking questions.

> **Order matters:** always start the Flask backend before the Streamlit frontend. The frontend will show API errors if Flask is not reachable.

## API Reference

All endpoints are prefixed with `/api`. The backend uses server-side sessions — the browser or `requests.Session` must forward the session cookie on every request.

### Authentication

| Method | Endpoint | Body | Description |
|---|---|---|---|
| `POST` | `/api/auth/register` | `{username, password}` | Create a new account. Password min 6 chars. |
| `POST` | `/api/auth/login` | `{username, password}` | Log in and set session cookie. |
| `POST` | `/api/auth/logout` | — | Clear session. Requires auth. |
| `GET` | `/api/auth/me` | — | Return current user info (no password hash). Requires auth. |

### Threads

All thread endpoints require authentication. Users can only access their own threads.

| Method | Endpoint | Body | Description |
|---|---|---|---|
| `GET` | `/api/threads` | — | List all threads for current user, newest first. |
| `POST` | `/api/threads` | `{title, tags[]}` | Create a new thread. |
| `GET` | `/api/threads/:id` | — | Get thread + full query history. |
| `PUT` | `/api/threads/:id` | `{title?, tags?[]}` | Update thread title and/or tags. |
| `DELETE` | `/api/threads/:id` | — | Delete thread and all its queries. |

### Queries

| Method | Endpoint | Body | Description |
|---|---|---|---|
| `GET` | `/api/threads/:id/queries` | — | List all queries in a thread, oldest first. |
| `POST` | `/api/threads/:id/queries` | `{question}` | Run guardrails → RAG → persist and return answer + sources. |

#### Query response (success `201`)

```json
{
  "_id": "664abc...",
  "thread_id": "664def...",
  "user_id": "664xyz...",
  "question": "What are GitLab's core values?",
  "answer": "GitLab's core values are...",
  "sources": ["https://handbook.gitlab.com/handbook/values/"],
  "created_at": "2024-06-01T10:23:45.123456+00:00"
}
```

#### Query response (guardrail block `422`)

```json
{
  "error": "Query contains content that is not allowed.",
  "code": "INJECTION_DETECTED",
  "blocked": true
}
```

## Guardrails System

Every query passes through `guardrails.py` **before** the vector store or main LLM is ever called. This keeps costs down and prevents misuse.

### Flow

```
query
  │
  ▼
Layer 1 (logical) ──── BLOCK ──► return error, skip everything
  │ PASS
  ▼
Layer 2 (prompt)  ──── BLOCK ──► return error, skip everything
  │ PASS
  ▼
ask_handbook() → ChromaDB → Gemini → answer
```

### Tuning thresholds via `.env`

```env
GR_MIN_LENGTH=3       # minimum characters
GR_MAX_LENGTH=1500    # maximum characters
GR_MIN_WORDS=1        # minimum word count
GR_MAX_WORDS=200      # maximum word count
GR_MODEL=gemini-2.0-flash   # model used for prompt guardrail
GR_TEMP=0.0           # temperature for guardrail classifier
```

## Streamlit Frontend

The frontend is a modular Streamlit application split across clearly separated layers so each file has a single responsibility.

## Data Flow

End-to-end flow for a single user question:

```
1. User types question in Streamlit chat input
2. streamlit_app/views/chat_view.py  →  services/api_client.create_query()
3. POST /api/threads/:id/queries  (Flask, main.py)
4. guardrails.run_guardrails(query, llm)
   ├── check_logical_guardrails()   — instant regex + length checks
   └── check_prompt_guardrails()    — Gemini 2.0 Flash classifier call
5. rag.ask_handbook(query)
   ├── vectorstore.similarity_search(query, k=5)   — ChromaDB + BGE embeddings
   ├── prompt | llm | StrOutputParser()             — Gemini 2.5 Flash
   └── extract source URLs from doc metadata
6. MongoDB: insert query document, update thread.updated_at
7. Flask returns 201 JSON { answer, sources, ... }
8. Streamlit: reload thread, re-render chat history with new bubble pair
```


**MongoDB authentication errors**

If your MongoDB instance requires a username/password, update `MONGO_URI`:
```env
MONGO_URI=mongodb://username:password@localhost:27017/
```
